# rpc

Main thread에서 Runtime으로 보내는 command에 **요청-응답(RPC) semantics**를 부여하는 얇은 브리지 레이어입니다.

## 역할

- `commandId` 발급·등록·타임아웃·정산
- RPC command(`CONNECT`, `PING`, `SUBSCRIBE`, `SEND_MESSAGE`)는 `Promise`로 반환
- post command(`UNSUBSCRIBE`, `DISCONNECT`)는 즉시 dispatch
- Worker push 응답을 pending RPC와 매칭

## 파일

| 파일 | 책임 |
|------|------|
| `index.ts` | rpc 모듈 public export |
| `create-socket-bridge.ts` | `SocketBridge` 팩토리 — `invoke` / `post` / `handleResponse` / `dispose` |
| `pending-rpc.ts` | `commandId` → `{ resolve, reject, timeoutId }` 맵 관리 |
| `settle-rpc-from-response.ts` | `ResponseEnvelope`을 보고 pending RPC resolve/reject |
| `create-socket-bridge.test.ts` | RPC 타임아웃·정산·rejectAll 동작 테스트 |

## `SocketBridge` 흐름

```text
invoke("SUBSCRIBE", { topic })
  → commandId 생성
  → pending.register(commandId, …)
  → dispatch(CommandEnvelope)
  → (응답) handleResponse → settleRpcFromResponse → Promise resolve

post("UNSUBSCRIBE", { topic })
  → dispatch만 수행 (Promise 없음)
```

## 설계 포인트

- `dispatch`는 `runtime.dispatch`를 주입받아 transport와 분리
- `onPush`로 RPC가 아닌 push(`MESSAGE`, `CONNECTION_STATE` 등)를 `SocketPushHandler`에 전달
- 연결 해제·runtime 교체 시 `rejectAll`로 dangling Promise 정리
