# react/hooks

React 컴포넌트에서 `SocketManager`를 구독·제어하는 hooks입니다.

## 파일

| 파일 | 책임 |
|------|------|
| `useClientSocketManager.tsx` | 클라이언트 전용 Provider + 싱글톤 manager lazy 생성 |
| `useInitSocketConnect.ts` | 마운트 시 `connect` / 언마운트·토큰 변경 시 `disconnect` |
| `useSocketConnection.ts` | 연결 상태·토픽 readiness를 `useSyncExternalStore`로 구독 |
| `useSocketSubscribe.ts` | `useSocketSubscription` — 토픽 구독·콜백·자동 cleanup |
| `useSocketForegroundReconnect.ts` | 포그라운드 복귀·watch·transport failure에 따른 복구 orchestration |

## Hook 역할 요약

### `useInitSocketConnect(accessToken, createConnectConfig)`

- 토큰이 있을 때만 connect
- `createConnectConfig`로 `SocketConfig` 비동기 생성 (앱 auth 연동)
- 토큰 무효화 시 disconnect

### `useSocketConnection` / `useSocketConnectionState` / `useSocketTopicsReady`

- `SocketSnapshotStore` 기반 파생 상태
- UI에서 연결 중·구독 준비 완료 여부 표시용

### `useSocketSubscription(topic, onMessage, options?)`

- `manager.subscribe` 래핑
- topic·handler 변경 시 재구독
- unmount 시 listener 제거

### `useSocketForegroundReconnect`

- `foreground-visibility-tracker`, `socket-connection-watch`, recovery runner 조합
- 백그라운드 장시간 체류·probe 실패·heartbeat lost 시 `checkConnectionState` 트리거
- transport failure debounce·recovery cooldown 적용
