# runtime

STOMP command를 실제로 전달하는 **실행 환경**을 추상화합니다. SharedWorker, Dedicated Worker, Main Thread 중 하나를 선택하고 실패 시 fallback chain을 탑니다.

## Runtime 모드

| Mode | 설명 |
|------|------|
| `shared` | `SharedWorker` + 단일 STOMP hub (탭 간 공유) |
| `dedicated` | 탭별 `DedicatedWorker` |
| `main` | Worker 미지원 환경(SSR 등)에서 메인 스레드 STOMP |

## Fallback chain (기본)

```text
shared → dedicated → main
```

`SocketRuntimeFallbackController`가 transport 실패 시 다음 모드로 교체합니다.
