# runtime/dispatch

Runtime으로 나가는 command를 **우선순위 큐**에 넣고, 동일 토픽의 중복 command를 coalesce합니다.

## 역할

- `CONNECT` / `PING` / `DISCONNECT`를 최우선 처리
- `SUBSCRIBE` / `SEND_MESSAGE`는 토픽 단위로 대기 중인 동일 command 취소 후 최신만 유지
- transport 미준비 시 큐에 보관, ready 시 flush

## 파일

| 파일 | 책임 |
|------|------|
| `index.ts` | dispatch 모듈 export |
| `priority-dispatch-queue.ts` | `PriorityDispatchQueue` — enqueue / setReady / clear |
| `extract-topic.ts` | command envelope에서 topic 문자열 추출 |
| `priority-dispatch-queue.test.ts` | 우선순위·coalescing·cancelled 반환 테스트 |

## 우선순위

| Priority | Command types |
|----------|---------------|
| 0 (최고) | `DISCONNECT`, `CONNECT`, `PING` |
| 1 | `SUBSCRIBE` |
| 2 | `UNSUBSCRIBE` |
| 3 (최저) | `SEND_MESSAGE` |

## `DispatchResult`

`enqueue`는 취소된 envelope 목록(`cancelled`)을 반환합니다. RPC 브리지가 해당 `commandId`의 pending Promise를 reject합니다.
