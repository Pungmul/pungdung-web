# protocol

메인 스레드와 Worker(또는 Main-thread runtime) 사이의 **메시지 계약**을 정의합니다. 비즈니스 도메인 타입이 아니라 전송 계층 envelope·상태 타입만 포함합니다.

## 역할

- STOMP 연결 설정(`SocketConfig`)과 리스너 인터페이스(`SocketListener`)
- Command / Response envelope 타입과 RPC vs fire-and-forget 구분
- 연결·구독·발행 상태 페이로드
- PING/PONG probe용 `PongPayload` (transport·heartbeat 진단)

## 주요 타입

### Command (Main → Runtime)

| `CommandType` | RPC 여부 | 설명 |
|---------------|----------|------|
| `CONNECT` | RPC | STOMP 연결 시작 |
| `PING` | RPC | transport·STOMP 생존 probe |
| `SUBSCRIBE` | RPC | 토픽 구독 |
| `SEND_MESSAGE` | RPC | 메시지 발행 |
| `UNSUBSCRIBE` | post | 구독 해제 (fire-and-forget) |
| `DISCONNECT` | post | 연결 종료 |

### Response (Runtime → Main)

| `ResponseType` | 설명 |
|----------------|------|
| `CONNECTED` | CONNECT 성공 |
| `PONG` | PING 응답 + probe 스냅샷 |
| `SUBSCRIBED` | 구독 완료 |
| `MESSAGE` | inbound STOMP 메시지 |
| `CONNECTION_STATE` | 연결 phase 변경 |
| `SUBSCRIPTION_STATE` | 토픽별 구독 상태 |
| `PUBLISHED` | 발행 결과 |
| `ERROR` / `RETRYING` | 오류·재시도 알림 |
